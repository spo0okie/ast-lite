Персонализация конфига под конкретную организацию обеспечена наличием префикса org1_ во всех контекстах, переменных и именах транков


Обеспеченные функции:
Дублирование вызова на второй номер
	для добавления дублера но локальный номер нужно объявить переменную Dubler/org1_num в которой указать куда должен быть отправлен вызов
	для вызова номера с задержкой нужно указать переменную DubDelay/org1_num (не более 99)

версия 1.7 - все персональные файлы вынесены в отдельный репозиторий ast.lite.sample, т.е. при выполнении git pull все должно работать без проблем. никакие персональные настройки не будут затронуты
версия 1.6 - добавлен макрос RecordCall для записи звонков, чтобы звонки записывались нужно установить переменную RecordCalls_<orgName> в какоенибудь значение
версия 1.5 - org1_api_call - отдельный контекст исходящих вызовов геренируемы call файлами. нужен чтобы отличать такие вызовы от прочих
		поскольку при таком исходящем генерируется сначала входящий вызов, как будто это нам звонят. такое событие можно ошибочно посчитать за входящий вызов
		буквально через час выяснилось, что контекст нам никак не поможет обнаружить такие вызовы, но я решил оставить его. это неплохо разделять вызовы по разным
		контекстам. может потом пригодится в целях безопасности
версия 1.4 - org1_incoming перенесен в приватную секцию, поскольку во первых там идет перенаправыление на голосовое меню, что нужно не везде
		во вторых там добавил обработку входящих CALLERID для приведения к одному виду
версия 1.3 - добавлена функция подмена CallerID (может пригождаться для сокрытия своего номера, например при звонке с номера директора отображается секретарь)
		но! она добавлена в месте локального вызова местного номера, т.е. не будет работать при вызове другого сервера, что не есть то что нам надо.
		НАДО ИСПРАВИТЬ!
версия 1.2 - добавление возможности обслуживания разных организаций в одном сервере:
		имена СИП клиентов необходимо снабжать префиксами организаций org1_ org2_ и т.п.
версия 1.1 - конфиг абстрагирован от отдельной организации, имена контекстов и переменных используют префикс org1
версия 1.0 - базовый функционал
